Context for future Codex sessions (quick start, no full tree walk needed):

- Project: Rust WASM formant/pitch analyzer with a web front-end under `webapp/`.
- Front-end is now TypeScript. Entry points: `webapp/src/main.ts` (UI/audio/wasm wiring) and `webapp/src/formantWorker.ts` (module worker for formant + pitch). WASM bindings live in `webapp/pkg/` (generated by wasm-pack; imported as ESM).
- Tooling: `webapp/package.json` uses ESM with scripts:
  - `npm run build` → `scripts/build.mjs` compiles `tsconfig.json` + `tsconfig.worker.json`, copies `index.html` and `pkg/` into `webapp/dist/`.
  - `npm run typecheck` → TS noEmit.
  - `npm run serve` → static server from `dist/` (defaults to PORT env or 5173; may need `PORT=3000` if sandbox blocks 5173).
- Ignored: `webapp/dist/` and `webapp/node_modules/` (see `webapp/.gitignore`). Keep `dist/` untracked; rebuild after TS changes.
- HTML loads `main.js` from `dist/`; make sure to rebuild before manual testing. Allow microphone permissions in the browser to see the real-time spectrum; formant history draws even without mic if data exists.
